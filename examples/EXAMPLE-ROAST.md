# create-react-app — The Honest Version

## What This Actually Does

It generates a new React project with a pre-configured build system. That's it. It scaffolds. It doesn't optimize. It doesn't deploy. It doesn't make your app fast. It gives you a folder with some files in it and a working `npm start`. For that narrow purpose, it's excellent. For everything else the README implies, it's not.

## What This Definitely Does NOT Do

- Make your app production-ready (the README says it's "optimized for production" — the build output is 2.4MB of unoptimized JS)
- Teach you React (the README links to the React docs, which is like a car manual linking to "what is a road")
- Configure anything beyond the absolute basics (you'll eject within 3 weeks, we both know this)

## Why This Exists

Because configuring Webpack made people cry. Create React App solved a real problem in 2016. It's still here because inertia is powerful and the React team blesses it. The real question isn't "why does this exist" but "why are you still using it in 2026 when Vite exists."

## Why You Should NOT Use This

- Vite is faster, simpler, and not abandoned
- The `react-scripts` dependency is a black box you'll fear ejecting from
- Build times grow exponentially with project size
- You'll end up learning Webpack anyway when something inevitably breaks

## Alternatives That Are Probably Better

- **Vite** — Faster, leaner, actively maintained. What CRA wanted to be.
- **Next.js** — If you need routing, SSR, or any feature beyond "hello world."
- **Parcel** — Zero-config that actually means zero-config.

## The Real Requirements

Node.js 14+. npm 6+. Patience. A willingness to accept that `react-scripts` will consume 600MB of disk space and you'll never know exactly what it contains. The courage to one day `eject` and face the Webpack config you've been hiding from.

## Bus Factor: 1

The project is maintained by a single team at Meta. If that team gets reorged — and they always get reorged — the bus has arrived.

## Most Egregious Lie

**"No configuration required."** You can configure nothing because everything is hidden inside `react-scripts`. When you inevitably need to customize something, you either eject (good luck) or use `react-app-rewired` (an unofficial hack maintained by a stranger). "No configuration" really means "no configuration possible."
